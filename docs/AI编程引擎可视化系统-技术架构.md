
# 一、技术选型

前端框架：

Vue3

语言：

TypeScript

构建工具：

Vite

包管理：

pnpm

运行方式：

Browser Server（BS）

开发工具：

VSCode

AI Coding：

Claude Code

语音输入：

系统语音转文字插件

浏览器：

Chrome

---

# 二、工程结构

建议结构如下：

docs/

存放需求文档。

src/

项目源码。

components/

公共组件。

views/

页面。

assets/

图片资源。

styles/

样式文件。

generated/

Claude Code 自动生成内容。

App.vue

项目入口。

---

# 三、开发流程

创建 Vue3 空项目。

保证项目可以运行。

Claude Code 每次根据新的需求修改代码。

修改完成以后：

自动保存。

运行。

浏览器刷新。

孩子立即体验。

继续下一轮修改。

---

# 四、Claude Code 工作方式

Claude Code 每次开始工作时：

首先阅读 docs 中的需求文档。

然后根据当前需求修改代码。

修改结束以后：

保证：

npm install

npm run dev

均可以正常运行。

---

# 五、修改原则

每次只完成一个需求。

不要一次修改多个功能。

不要修改项目结构。

不要修改 Vite 配置。

不要修改 package.json。

不要增加复杂第三方依赖。

不要新增后端。

整个项目保持单页面。

---

# 六、页面规范

始终保持：

一个页面。

简单布局。

功能集中。

所有新增内容都放到游戏区域。

不要增加复杂导航。

不要增加登录。

不要增加后台。

---

# 七、代码规范

使用 Composition API。

使用 TypeScript。

组件职责单一。

变量命名清晰。

删除无用代码。

不要保留 TODO。

所有代码必须真正可运行。

---

# 八、AI Coding 规范

每次收到新的需求：

理解需求。

分析当前代码。

找到最小修改方案。

完成代码修改。

保持已有功能。

不要覆盖历史功能。

如果新增功能与已有功能冲突：

优先兼容。

---

# 九、项目目标

整个项目最终形成：

一个可以不断成长的软件。

每一句自然语言。

都对应一次真正的软件升级。

整个项目重点不是代码量。

而是展示：

自然语言

↓

AI Coding

↓

浏览器运行

↓

软件成长

这一完整闭环。

---

# 十、后续扩展

后续可以继续扩展：

更多小游戏。

更多动画。

更多 AI 能力。

更多交互方式。

但始终保持：
