
# 一、项目简介

## 项目名称

一句话，创造一个小游戏

## 技术代号

AI编程引擎可视化系统（AI Coding Playground）

## 项目背景

本项目用于参加 AI 亲子活动。

整个项目希望展示一种新的 AI Coding 开发模式。

传统的软件开发需要程序员编写代码，而本项目希望展示：

> 孩子负责表达创意，AI负责编程，程序员负责审核和运行。

整个开发过程完全采用自然语言驱动，由孩子一句一句描述需求，AI 根据需求不断修改代码，最终生成一个真正可以运行的网页小游戏。

整个过程比最终作品更重要。

项目重点展示：

-   AI Coding
    
-   自然语言编程
    
-   软件实时演进
    
-   人与 AI 协同开发
    

而不是展示最终做出了什么小游戏。

---

# 二、项目目标

通过一个可运行的 Vue3 项目，让孩子像"产品经理"一样提出自己的想法。

例如：

我要一个小恐龙。

我要它会飞。

我要它会喷火。

我要它生活在糖果世界。

每一句话都会交给 Claude Code。

Claude Code 根据已有工程不断修改代码。

浏览器实时刷新。

孩子立即看到自己的想法变成真正的软件。

---

# 三、项目定位

本项目不是 AI 生图。

不是 AI 绘本。

不是 AI 聊天。

而是：

AI Coding 实时开发演示系统。

整个项目重点展示：

孩子一句话

↓

AI 修改代码

↓

浏览器运行

↓

继续修改

↓

软件不断成长

---

# 四、项目流程

## 第一步

创建空项目。

项目可以正常运行。

首页只有一句欢迎语。

## 第二步

孩子开始提出第一个需求。

例如：

我要一个小恐龙。

Claude Code 修改项目。

浏览器刷新。

出现小恐龙。

## 第三步

继续增加需求。

例如：

我要一个喂食按钮。

Claude Code 修改。

浏览器刷新。

出现按钮。

## 第四步

继续增加需求。

例如：

我要它会飞。

Claude Code 修改。

浏览器刷新。

小恐龙可以飞。

整个项目一直保持：

说一句

修改一次

运行一次

---

# 五、页面设计

整个项目只有一个页面。

页面尽量简单。

页面包含：

① 页面标题

AI Coding Playground

② 游戏区域

Claude Code 生成的内容

③ 当前状态

例如：

运行成功

修改次数

最后一次修改时间

④ 修改日志

显示最近几次孩子说的话。

例如：

新增小恐龙

增加飞行能力

增加草莓食物

增加彩虹背景

---

# 六、项目原则

整个项目始终保持：

简单

稳定

可运行

每次只增加一个需求。

每次修改以后必须立即可以运行。

不允许因为增加一个功能导致整个项目无法启动。

---

# 七、Claude Code 工作原则

Claude Code 不是重新开发整个项目。

而是在已有项目基础上持续修改。

每次：

只完成一个需求。

不要删除已有功能。

不要重构整个项目。

保持所有已有功能继续可用。

---

# 八、孩子参与方式

孩子不用学习编程。

孩子只需要说：

我要……

它可以……

它不要……

它应该……

所有代码由 AI 自动完成。

孩子真正参与的是：

创意。

想法。

规则设计。

整个过程让孩子理解：

软件也是一点一点成长出来的。

---

# 九、演示流程

启动项目。

浏览器打开。

孩子说一句话。

Claude Code 修改。

浏览器刷新。

继续体验。

继续修改。

最后形成一个真正可以玩的小游戏。

整个活动重点展示：

软件是如何一步一步成长的。

而不是一次性生成。
