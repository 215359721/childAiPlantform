## 💡 这是什么

**AI Coding Playground** 是一个面向亲子 AI 活动的演示项目，它展示了一种**全新的软件开发模式**：

> 把孩子从"软件的使用者"变成"软件的创造者"——而且，**一行代码都不用写**。

孩子在网页上用自然语言一句一句提需求（"我要一个小恐龙""我要它会飞""我要它会喷火"），网页通过本地接口把每一句话送给 **Claude Code**，由 AI 在已有工程上做**最小修改**，Vite 热更新让浏览器自动刷新——软件就这样**一句话、一句话地长大**。

它不是一个"一次性生成成品"的工具，而是一场**看得见的、实时的、人机协同的编程过程**。

---

## 🌱 一个新的开发模式

传统软件开发，是程序员把需求翻译成代码。
而这里展示的模式，是三种角色各司其职：

| 角色 | 职责 | 需要会写代码吗 |
| --- | --- | --- |
| 👧 **孩子** | 提创意、定规则、做"产品经理" | ❌ 完全不用 |
| 🤖 **AI（Claude Code）** | 把每一句话变成能跑的代码 | ✅ 全自动 |
| 👨‍💻 **程序员** | 审核改动、保证项目始终能运行 | ✅ 只做审核 |

孩子说的是**想法**，AI 写的是**代码**，程序员做的是**把关**。
三方协作，软件在对话中成长。

## 🎯 它不是什么

为了说清楚这个产品，先说它**不是**什么：

- ❌ 不是 **AI 生图**——画的不是图，是真正能交互的网页程序
- ❌ 不是 **AI 绘本**——产出的是可运行、可继续修改的软件
- ❌ 不是 **AI 聊天**——每一句话都对应一次**真实的代码升级**

它是 👉 **AI Coding 实时开发演示系统**：让"自然语言"真正驱动"软件成长"。

## 🔄 软件是怎么"长大"的

整个过程是一个完整的闭环，重点在于**过程**，而非最终作品：

```

 ──────────────── ▶   ──────────────────── ▶   ──────────── ▶   ────────────────────
 "我要一个会飞的恐龙"      只做最小改动、保留已有功能        自动重新编译            孩子立刻看到结果
        ▲                                                                        │
        │                                                                        ▼
        └──────────────────────  软件一点点长大，继续说下一句  ◀──────────────────────
```

每一句自然语言 → 都对应一次真正的软件升级。

## 🚀 快速开始

### 运行前提

1. 本机已安装 **[Claude Code](https://docs.anthropic.com/en/docs/claude-code)**（终端命令 `claude`）并**已登录**。
   网页输入框会把孩子的话送给它。鉴权**复用 `claude` 的登录态（OAuth），无需单独的 API Key**。
2. Node ≥ 18，[pnpm](https://pnpm.io/)。

### 三条命令

```bash
pnpm install      # 安装依赖（首次）
pnpm run dev      # 启动开发服务器，浏览器自动打开 ← 演示请用这条
pnpm run build    # 类型检查 + 构建
```

> ⚠️ 演示请始终用 `pnpm run dev`。`pnpm run preview` 预览的是构建产物，**没有 AI 接口**。

## 👨‍👩‍👧 怎么玩（亲子演示流程）

1. 终端运行 `pnpm run dev`，浏览器自动打开糖果色页面。
2. 孩子在输入框打字（或点 🎤 用语音说），点 **✨ 帮我做**。
3. 页面实时显示 AI 的工作过程（想办法 → 改代码 → 完成）。
4. Claude 改好代码后，游戏区和修改日志自动"长大"，孩子立刻看到结果。
5. 继续说、继续改，软件不断成长。

一次典型的亲子对话，可能长这样：

> 🧒 *我要一个小恐龙。* → 🦕 画布上出现一只小恐龙
> 🧒 *我要它会飞。* → 🕊️ 恐龙开始拍翅膀飞起来
> 🧒 *我要它生活在糖果世界。* → 🍭 背景变成糖果色、飘起彩色圆斑
> 🧒 *给它加一个喂食按钮。* → 🍓 出现按钮，点了能喂草莓

> 单次请求约 5～20 秒，取决于 Claude 的回复速度。

### 💾 本地作品库（右上角 📁）

做到一半的好作品，可以点右上角 **📁** 选「另存为」（起个名字），然后新建别的继续做；随时从作品库切回来接着让 AI 改。作品只存在你这台电脑的 `works/` 目录——**不上传、不进 git、不需要账号**，关掉浏览器也不会丢。

## ❤️ 为什么"过程"比"作品"更重要

大多数 AI 工具追求"一键生成完美成品"。但对孩子来说，真正珍贵的不是最后那只飞起来的恐龙，而是他们亲眼看到：

- **想法是可以变成现实的**——我说的每一句话，软件都认真地回应了；
- **软件不是天生的，是一点点长大的**——和种一棵树、养一只小动物一样；
- **创造是有节奏的**——先要一个，再加一个，慢慢变丰富。

这是一种属于这个时代的**创造启蒙**：孩子学会的不是代码，而是**用语言去塑造世界**的信心。

## 🛠️ 技术栈

| 维度 | 选型 |
| --- | --- |
| 前端框架 | Vue 3（Composition API + `<script setup>`） |
| 语言 | TypeScript |
| 构建工具 | Vite（热更新即"软件长大"的体验基础） |
| 包管理 | pnpm |
| AI 引擎 | 本机 [Claude Code](https://docs.anthropic.com/en/docs/claude-code)（`claude -p`） |
| 语音输入 | 浏览器原生 Web Speech API（零依赖、零 Key） |
| 语音播报 | 浏览器原生 SpeechSynthesis（纯前端） |
| 后端 | 无独立后端——dev 下由 Vite 插件暴露本地接口（`/api/wish` 改代码、`/api/works/*` 本地作品库） |

**核心设计：单页面、本地接口、零额外依赖、零 API Key。**

## ❓ 常见问题

**Q：需要联网吗？**
需要。Claude Code 本身需要联网调用模型，但网页与 `claude` 之间的通信走本机 localhost。

**Q：需要单独申请 API Key 吗？**
不需要。直接复用 `claude` 命令的登录态（OAuth）。

**Q：孩子的话安全吗？**
安全。孩子的文字通过 **stdin** 送给 `claude`，命令行参数是固定的；网页与 `claude` 仅在本机通信，不经过任何外部服务器。

**Q：多大孩子适合？**
认识字、能表达"我要……它会……"的小朋友即可。不认识字也没关系——点 🎤 用语音说。

**Q：AI 会不会把项目改崩？**
不会。CLAUDE.md 里的"最小修改契约"约束 AI 每次只在 `src/generated/` 下"只增不改"，状态与计数从日志自动派生。万一改乱，右上角「🔄 重新开始」一键还原；做好的作品也可以先用「📁 作品库」存起来，随时切回好版本。
